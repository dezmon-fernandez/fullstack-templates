# .agents/

Self-contained workspace for AI-assisted development. Keeps AI-generated artifacts and AI-consulted context out of the main source tree so the codebase stays clean.

## Layout

```
.agents/
├── PRD.md          # Product Requirements Document — source of truth (generated by /create-prd)
├── plans/          # Per-feature implementation plans (generated by /generate-plan, consumed by /execute-plan)
│   └── *.md
└── reference/      # Curated reference material the AI consults
                    # Architectural decisions, stack quirks, patterns to preserve,
                    # cross-cutting context that isn't obvious from code
```

## Workflow

1. `/create-prd` writes the PRD to `PRD.md`. This is the project's source of truth — required first step in a fresh repo.
2. `/generate-plan "<feature description>"` reads the PRD and writes a per-feature plan to `plans/<feature>.md`.
3. `/execute-plan plans/<feature>.md` implements the feature with tests and validation.

## Conventions

- **PRD lives at `PRD.md`** — authoritative for scope, architecture, success criteria. Update when MVP scope or architecture changes.
- **Feature plans live at `plans/<kebab-name>.md`** — one per feature, generated from PRD + feature description.
- **Reference material in `reference/`** should describe *why* things are the way they are — the stuff that's hard to recover from code alone.

## Notes

- `.next-docs/` (at repo root) is version-locked Next.js documentation, separate from this workspace.
